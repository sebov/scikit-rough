# Correctness of the Decision Bireduct Sampling Algorithm

The sampling algorithm always produces a valid decision bireduct, and every decision bireduct is
achievable by some choice of sampling attributes $A^\diamond$.

## Statement

Let $\mathbb{A} = (U, A \cup \{d\})$ and $A^\diamond \subseteq A$ be given. Then the outcome
$(X, B)$ of the sampling algorithm is a decision bireduct for $\mathbb{A}$. Moreover, each decision
bireduct for $\mathbb{A}$ can be obtained as a result of the algorithm.

## Background

The sampling algorithm works in three stages:

1. Draw a subset of attributes $A^\diamond \subseteq A$ (either passed as input or chosen
   randomly).
2. For each indiscernibility class $E \in U / A^\diamond$, pick one representative object
   $u^\diamond \in E$. Build $U^\diamond$ as the set of these representatives. Build $X$ as the
   union over all representatives of $E \cap X^{\langle d(u^\diamond) \rangle}$ -- i.e., all
   objects in $E$ that share the representative's decision value.
3. Compute an arbitrary decision reduct $B \subseteq A^\diamond$ for the reduced consistent table
   $\mathbb{A}_\diamond = (U^\diamond, A^\diamond \cup \{d\})$. Output $(X, B)$.

The method combines Monte Carlo attribute sampling with classical reduct computation, analogous to
bagging but with constraints tailored to bireduct induction.

## Proof

The proof has two parts.

### Part 1: The Output Is a Decision Bireduct

**Consistent sub-table.** Since $U^\diamond$ contains exactly one object from each
$A^\diamond$-indiscernibility class, the sub-table $\mathbb{A}_\diamond$ is consistent: within each
$A^\diamond$-class there is only one object, so trivially no two objects with the same
$A^\diamond$-values have different decisions.

**Functional dependency.** $B$ is a decision reduct for $\mathbb{A}_\diamond$, so $B \subseteq
A^\diamond$ and $B \Rrightarrow_{U^\diamond} d$. By construction, every object in $X$ has the same
attribute values on $A^\diamond$ (hence on $B$) and the same decision as some representative in
$U^\diamond$. Therefore $B \Rrightarrow_X d$.

**Attribute irreducibility.** $B$ is a reduct (irreducible) for $\mathbb{A}_\diamond$, so no proper
subset $B' \subsetneq B$ satisfies $B' \Rrightarrow_{U^\diamond} d$. Since $U^\diamond \subseteq
X$, irreducibility carries over to $X$: if $B' \Rrightarrow_X d$, then in particular $B'
\Rrightarrow_{U^\diamond} d$, which contradicts the reduct property.

**Object maximality.** We cannot extend $X$ with any $u \in U \setminus X$. Let $u \in U
\setminus X$ and let $E$ be its $B$-indiscernibility class (or even $A^\diamond$-class). Since
$u \notin X$, it must be that $d(u)$ differs from the decision of the representative chosen for
$E$. Hence $E \cap X$ consists exclusively of objects with a different decision from $d(u)$.
Adding $u$ to $X$ would violate the functional dependency, since $u$ would share $B$-values with
objects in $E \cap X$ but have a different decision.

### Part 2: Every Decision Bireduct Is Achievable

Let $(X, B)$ be an arbitrary decision bireduct. Set $A^\diamond = B$. Since $(X, B)$ is a bireduct,
$X$ has at least one object in each $B$-indiscernibility class. Form $U^\diamond$ by picking exactly
one object from $X$ per $B$-indiscernibility class.

Since $B$ defines a decision reduct for $\mathbb{A}_\diamond$ (each representative from a distinct
$B$-class, consistent sub-table) and $B$ is irreducible for $X$, it is also irreducible for
$U^\diamond \subseteq X$. Thus $B$ is the only decision reduct for $\mathbb{A}_\diamond$, and the
algorithm recovers exactly $(X, B)$.

## Remarks

This result connects to the decision rules perspective (`prop-bireduct-objects-and-rules`): the
reduced table $\mathbb{A}_\diamond$ is a compact representation of if-then rules generated by
attributes in $B$, with the sum of their supports equalling $X$.

In the $\gamma$-case (`prop-gamma-decision-bireduct-sampling`), the construction is simpler because
the object set is always the full positive region $POS_{\mathbb{A}}(B)$.
